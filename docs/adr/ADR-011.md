# ADR-011: Documentation Reconciliation Skill

## Status
Proposto

## Date
2026-07-05

## Contexto

### Problema
Documentação canônica (README, CHANGELOG, USAGE) e específica (ADRs, BP, TODOs) frequentemente ficam desatualizadas. Gaps incluem:
- Status incorretos em ADRs
- ADRs implementadas não arquivadas
- Changelog desorganizado
- README desatualizado

### Motivação
Precisamos de uma skill automatizada que audite e reconcilie documentação contra realidade do código, seguindo o padrão Ultra-High Quality Grade.

### Restrições
- Skill deve seguir padrão ADR-006
- Templates separados para audit-report e reconciliation-checklist
- Cross-references com governance e adr-generator

## Decisão
Criar skill `documentation-reconciliation` com workflow de 8 fases que audita e reconcilia documentação canônica e específica.

## Alternativas Consideradas

### Alternativa A: Manual reconciliation
Auditar manualmente sempre que necessário.
- Prós: Simples, sem automação
- Contras: Propenso a erros, não padronizado, esforço repetitivo

### Alternativa B: Skill automatizada (escolhida)
Criar skill com workflow estruturado e templates.
- Prós: Padronizado, acionável, reutilizável, auditação consistente
- Contras: Esforço inicial de criação

## Consequências

### Positivas
- Documentação sempre sincronizada
- Gaps identificados automaticamente
- Processo de arquivamento padronizado
- Compliance score mensurável

### Negativas
- Skill adicional a manter
- Templates extras no repositório

# ADR-009: Resolução de Débitos da Auditoria v2.1.0

## Status
Proposto

## Data
2026-07-05

## Contexto

A auditoria v2.1.0 (A- — 94/100) identificou débitos estruturais e recomendações pendentes que precisam ser resolvidos para elevar a nota para 95+ e garantir maturidade completa do runtime de governança.

**Débitos identificados:**
- Validação de consistência version README ↔ index.json ausente no CI
- `governance/SKILL.md` não reconhece modo "solo + agentes" como colaboração válida
- `agent-orchestration/SKILL.md` cita "Claude Opus 3" (nomenclatura desatualizada)
- Exemplos de skills com viés de stack (majoritariamente Node/npm)

**Fixes já implementados nesta iteração:**
- `adr-generator` agora cria BP + TODO junto com ADR (regra obrigatória)
- `AGENTS.md` documenta ciclo completo: ADR → Implementação → Arquivamento → Deploy gh-pages

## Decisão

Implementar as recomendações da auditoria em 3 fases priorizadas:

1. **Quick wins (CI):** Adicionar validação de version sync no `validate-index.sh`
2. **Médio (skills):** Atualizar `governance` e `agent-orchestration` para refletir modos reais de uso
3. **Estrutural (documentação):** Expandir exemplos para múltiplas stacks e documentar ciclo de vida no AGENTS.md

## Alternativas Consideradas

- **Alternativa A: Implementar tudo de uma vez**
  - Prós: Resolução completa em uma iteração
  - Contras: Escopo grande, risco de regressão

- **Alternativa B: Priorizar quick wins e médio, deferir estrutural (escolhida)**
  - Prós: Reduz risco, entrega valor imediato, mantém foco
  - Contras: Exemplo de stack permanece como débito menor

- **Alternativa C: Apenas quick wins**
  - Prós: Mínimo esforço
  - Contras: Não resolve gaps de documentação

## Consequências

### Positivas
- CI valida consistência version automaticamente
- Skills refletem modos reais de uso (solo + agentes)
- Nomenclatura atualizada para modelos de IA atuais
- Ciclo de vida ADR documentado em AGENTS.md

### Negativas
- Exemplos multi-stack ficam como débito futuro (não bloqueante)

### Riscos
- Mudanças em `validate-index.sh` podem causar falsos positivos → mitigado com testes

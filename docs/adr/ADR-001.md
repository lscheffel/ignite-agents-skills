# ADR-001: Consolidar registry de skills em único index.json

## Status
Aceito

## Contexto
O repositório `ignite-agents-skills` serve como registry remoto de skills para agentes compatíveis com o padrão Agent Skills, utilizado pelo Kilo Code, OpenCode e outros agentes. Inicialmente, foi tentado seguir a convenção `.well-known/skills/` com manifestos em formato diferente, mas isso resultou em:

- **Duplicação de registry**: múltiplos arquivos JSON com informações semelhantes
- **Incompatibilidade com Kilo Code**: o mecanismo `skills.urls` do Kilo busca `{url}/index.json` e resolve arquivos como `{url}/{skill-name}/{file}`, onde `file` deve ser **relativo à pasta da skill**, não um path absoluto
- **Paths incorretos**: a tentativa anterior usava paths completos (`skills/nome/SKILL.md`) no manifesto, causando duplicação de path na resolução (`skills/nome/skills/nome/SKILL.md`)

## Decisão
Consolidar o registry em um único arquivo `skills/index.json` com paths relativos à pasta de cada skill, removendo manifestos redundantes na raiz e em `.well-known/`.

O `skills/index.json` é a **fonte única** e canônica, contendo:
- Lista de skills com `name` e `files` (paths relativos)
- Versão do registry
- Descrição do registry

## Alternativas Consideradas

- **Alternativa A**: Manter `.well-known/skills/` com paths absolutos
  - Prós: Segue convenção RFC 8615 para descoberta de recursos
  - Contras: Incompatível com mecanismo de resolução do Kilo Code, requer manutenção de múltiplos arquivos

- **Alternativa B**: Manter registry na raiz (`registry.json`)
  - Prós: Simples de localizar
  - Contras: Não segue convenção de skills do Kilo, paths precisariam ser absolutos

- **Alternativa C**: Consolidar em `skills/index.json` (escolhida)
  - Prós: Compatível com mecanismo `skills.urls` do Kilo, paths relativos funcionam corretamente, estrutura intuitiva
  - Contras: Quebra com convenção `.well-known/`

## Consequências

### Positivas
- Compatibilidade total com o mecanismo de resolução de arquivos do Kilo Code
- Single source of truth: apenas um arquivo a ser mantido
- Paths relativos funcionam corretamente (`SKILL.md` em vez de `skills/nome/SKILL.md`)
- Estrutura de diretórios alinhada com a URL de publicação (GitHub Pages)

### Negativas
- Não segue a convenção `.well-known/` recomendada para descoberta de recursos
- Quebra com tentativa anterior (arquivos removidos)

### Riscos
- **Risco**: Projetos já configurados com URLs antigas podem falhar
  - **Mitigação**: Documentação atualizada em README.md com instruções de configuração correta

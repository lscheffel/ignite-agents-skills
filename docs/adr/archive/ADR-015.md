# ADR-015: Fix Caminhos Relativos Depth-Aware no Build.py

## Status
Implementado

## Date
2026-07-06

## Contexto

### Diagnóstico

| Capacidade | Status | Evidência |
|------------|--------|-----------|
| Links relativos em HTMLs dinâmicos | ❌ | `readme.html` gerava `../../adr/ADR-001.html` (404) |
| Nav links depth-aware | ❌ | `get_page_template()` usava `../../` fixo para todas as páginas |
| ADR links funcionais | ❌ | Caminhos não resolviam para `/pages/adr/ADR-XXX.html` |

### Consequências da Lacuna

- Todos os links ADR no `readme.html` retornavam 404
- Nav links em `usage.html` apontavam para caminhos inexistentes
- Usuários não conseguiam navegar entre páginas

## Decisão

Implementar sistema de **caminhos relativos depth-aware** onde cada página calcula o prefixo correto baseado na sua profundidade no diretório:

| Página | Depth | Nav Prefix | Exemplo |
|--------|-------|------------|---------|
| `pages/readme.html` | 1 | *(vazio)* | `adr/ADR-001.html` |
| `pages/adr/ADR-XXX.html` | 2 | `../` | `../index.html` |
| `pages/skills/name/index.html` | 3 | `../../` | `../../adr/ADR-005.html` |
| `pages/skills/name/cat/file.html` | 4 | `../../../` | `../../../adr/ADR-001.html` |

### Implementação

```python
# convert_inline_md e convert_md_to_html
prefix = "../" * (depth - 1) if depth > 1 else ""

# get_page_template (nav links)
nav_prefix = "../" * (depth - 1) if depth > 1 else ""

# generate_skill_page → depth=3
# generate_file_page → depth=4
# generate_doc_page → depth=1
# generate_adr_page → depth=2
```

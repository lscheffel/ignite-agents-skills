# ADR-007: Skill para Geração de AGENTS.md Adaptativo

## Status

Implementado

## Contexto

O repositório `ignite-agents-skills` é uma coleção de skills para agentes de IA que seguem o padrão Ultra-High Quality Grade. Atualmente, não existe uma skill dedicada para gerar e manter o arquivo `AGENTS.md` na raiz dos projetos.

### Problema

1. **Ausência de AGENTS.md**: Projetos não possuem um arquivo de governança que instrua agentes de IA sobre como interagir com o código
2. **Falta de adaptabilidade**: O template existente em `repo-bootstrap` é genérico e não se adapta ao contexto específico do projeto
3. **Desalinhamento**: Agentes de IA podem tomar decisões que não estão alinhadas com as práticas do projeto
4. **Inconsistência**: Cada projeto pode ter práticas diferentes, mas não há um mecanismo para documentar e comunicar essas práticas aos agentes

### Motivação

- Agentes de IA precisam de contexto sobre o projeto para tomar decisões alinhadas
- O `AGENTS.md` deve ser um "conciliador" entre o estado atual do projeto e as regras de governança
- A skill deve ser adaptável: se o projeto é um CRM, o AGENTS.md deve refletir isso; se é um repositório de skills, deve refletir esse contexto

### Restrições

- Deve seguir o padrão Ultra-High Quality Grade
- Deve ser compatível com a estrutura existente de skills
- Deve ser validável pelo `validate-skill.sh`
- Deve ser registrado automaticamente no `index.json`

## Decisão

Criar uma nova skill `agents-md-generator` que:

1. **Detecta o contexto do projeto** automaticamente
2. **Gera AGENTS.md adaptativo** baseado no contexto detectado
3. **Mantém o arquivo atualizado** conforme o projeto evolui
4. **Segue o padrão Ultra-High Quality Grade**

### Estrutura da Skill

```
skills/agents-md-generator/
├── SKILL.md                    # Documentação principal
├── templates/
│   ├── AGENTS-base.md         # Template base
│   ├── AGENTS-skills-repo.md  # Template para repositórios de skills
│   ├── AGENTS-crm.md          # Template para projetos CRM
│   ├── AGENTS-api.md          # Template para projetos de API
│   └── AGENTS-webapp.md       # Template para webapps
├── examples/
│   ├── before-after.md        # Exemplos de antes e depois
│   └── context-detection.md   # Exemplos de detecção de contexto
└── checklists/
    └── validation.md          # Checklist de validação
```

### Funcionalidades Principais

#### 1. Detecção Automática de Contexto

A skill deve detectar automaticamente:
- **Tipo de projeto**: CRM, API, WebApp, Biblioteca, CLI, etc.
- **Tecnologias**: Linguagens, frameworks, bancos de dados
- **Padrões**: Arquitetura (Clean, Hexagonal, DDD), padrões de código
- **Governança**: Branching strategy, processo de PR, CI/CD
- **Equipe**: Tamanho, estrutura, papéis

#### 2. Geração de Conteúdo Adaptativo

O AGENTS.md gerado deve incluir:
- **Visão Geral**: Descrição do projeto e seu propósito
- **Estrutura**: Organização de diretórios e responsabilidades
- **Padrões**: Convenções de código, commits, branches
- **Comandos**: Scripts importantes para desenvolvimento
- **Governança**: Processos de revisão, aprovação, deploy
- **Skills Recomendadas**: Skills relevantes para o projeto
- **Anti-patterns**: O que evitar neste tipo de projeto
- **Edge Cases**: Situações especiais e como tratá-las

#### 3. Manutenção Automática

A skill deve:
- Detectar mudanças no projeto que afetem o AGENTS.md
- Sugerir atualizações quando o contexto mudar
- Validar se o AGENTS.md está atualizado

## Alternativas Consideradas

### Alternativa A: Template Estático

- **Descrição**: Usar o template existente em `repo-bootstrap` sem adaptação
- **Prós**: Simples, rápido de implementar
- **Contras**: Não se adapta ao contexto, genérico demais
- **Rejeitado por**: Não atende ao requisito de adaptabilidade

### Alternativa B: Skill Separada por Tipo de Projeto

- **Descrição**: Criar skills separadas (agents-md-crm, agents-md-api, etc.)
- **Prós**: Especialização por contexto
- **Contras**: Muitas skills, manutenção difícil, duplicação de código
- **Rejeitado por**: Viola o princípio DRY, difícil de manter

### Alternativa C: Skill Adaptativa Única (Escolhida)

- **Descrição**: Uma skill que detecta o contexto e gera o AGENTS.md apropriado
- **Prós**: Única fonte de verdade, adaptável, fácil de manter
- **Contras**: Mais complexa de implementar
- **Escolhido por**: Equilíbrio entre funcionalidade e manutenibilidade

## Consequências

### Positivas

- **Consistência**: Todos os projetos terão um AGENTS.md padronizado
- **Adaptabilidade**: O conteúdo se adapta ao contexto do projeto
- **Manutenibilidade**: Uma única skill para manter
- **Validação**: Pode ser validada pelo `validate-skill.sh`
- **Integração**: Registra automaticamente no `index.json`

### Negativas

- **Complexidade**: Detecção de contexto requer lógica adicional
- **Manutenção de Templates**: Múltiplos templates para manter
- **Curva de Aprendizado**: Usuários precisam aprender a usar a skill

### Riscos

- **Detecção Incorreta**: O contexto pode ser mal interpretado
  - **Mitigação**: Permitir override manual do contexto detectado
- **Templates Desatualizados**: Templates podem ficar obsoletos
  - **Mitigação**: Processo de revisão e atualização periódica
- **Complexidade Excessiva**: A skill pode ficar complexa demais
  - **Mitigação**: Começar com templates básicos e evoluir

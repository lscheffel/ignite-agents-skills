# ADR-013: Expansão do Build.py para Incluir ADRs e Referências

## Status
Implementado (emergencial)

## Data
2026-07-05

## Contexto

### Diagnóstico

| Capacidade | Status | Evidence |
|------------|--------|----------|
| Referências de ADRs em README/USAGE | ❌ | Links para `docs/adr/archive/ADR-XXX.md` não são renderizados como HTML |
| Documentos de referência | ❌ | Arquivos em `docs/` não são incluídos no build |

### Consequências da Lacuna

- Usuários não conseguem navegar para ADRs via GitHub Pages
- Links quebram quando acessados via HTML renderizado
- Perda de contexto arquitetural na versão HTML

## Decisão

Expandir `pages/build.py` para incluir:
1. **ADRs arquivadas** — `docs/adr/archive/*.md` → `pages/adr/*.html`
2. **Documentos de referência** — `docs/*.md` → `pages/docs/*.html`
3. **Regravação de links** — Converter `docs/adr/archive/ADR-XXX.md` em links relativos para `../../adr/ADR-XXX.html`

### Implementação Imediata

```
1. Adicionar função generate_adr_pages() no build.py
2. Adicionar função generate_docs_pages() no build.py
3. Atualizar convert_inline_md() para converter paths de ADRs
4. Adicionar nav link para "ADRs" no template
```

## Consequências

### Positivas
- 12 ADRs adicionais renderizados como HTML
- Navegação completa entre skills, ADRs e docs
- Links relativos corretos em todos os documentos

### Negativas
- +12 páginas HTML (impacto mínimo)
- Build script mais complexo

### Riscos
- **Risco**: Links internos podem não converter corretamente
  - **Mitigação**: Testar com `README.html` e `USAGE.html` antes do merge

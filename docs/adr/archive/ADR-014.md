# ADR-014: Fix Workflow sync-and-deploy — Sync Completo de master para gh-pages

## Status
Implementado

## Date
2026-07-06

## Contexto

### Diagnóstico

| Capacidade | Status | Evidência |
|------------|--------|-----------|
| Deploy gh-pages sincroniza todos os arquivos | ❌ | `--strategy=ours` ignorava arquivos novos do master |
| Links ADR funcionais no HTML | ❌ | `pages/adr/` não existia em gh-pages |
| Workflow CI detecta mudanças | ✅ | `sync-and-deploy.yml` rodava corretamente |

### Consequências da Lacuna

- ADRs linkadas no `readme.html` retornavam 404
- `pages/adr/` directory não existia em gh-pages
- Usuários não conseguiam navegar para ADRs via GitHub Pages

## Decisão

Substituir `git merge --strategy=ours` por uma abordagem de **full sync** que copia todos os arquivos do master para gh-pages:

### Implementação Imediata

```yaml
# ANTES (quebrado)
git merge origin/master --strategy=ours --allow-unrelated-histories --no-edit

# DEPOIS (funcional)
git ls-files | grep -v -E '^(\.git|\.nojekyll|CNAME)' | xargs git rm -rf
git checkout origin/master -- .
touch .nojekyll
git add -A
git commit -m "chore: full sync from master [skip ci]"
```

## Alternativas Consideradas

### Alternativa A: Usar `git merge` sem strategy
Manter merge mas sem `--strategy=ours`.
- Prós: Simples
- Contras: Pode criar conflitos

### Alternativa B: Usar rsync para copiar arquivos
Copiar arquivos via rsync em vez de git.
- Prós: Mais controle
- Contras: Mais complexo, perde histórico git

### Alternativa C: Full sync (escolhida)
Remover todos os arquivos e re-checkout do master.
- Prós: Garante que gh-pages espelha master 100%
- Contras: Histórico de commits do gh-pages menos limpo

## Consequências

### Positivas
- gh-pages espelha master 100%
- Todos os links funcionais (ADRs, skills, templates)
- Workflow CI agora sincroniza corretamente

### Negativas
- gh-pages perde histórico de commits anteriores
- Commits de sync são mais "poluídos"

### Riscos
- **Risco**: `.nojekyll` pode ser removido acidentalmente
  - **Mitigação**: Workflow re-cria `.nojekyll` após sync
